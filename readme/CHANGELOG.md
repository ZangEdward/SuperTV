## [5.5.24.523] - 2026-05-20

### 新增
- **多线程CPU优化 v2**：将 M3U8 并发下载线程数提升至 16 线程（视设备性能动态平衡），深度优化并发调度，大幅提高下载速度同时降低 CPU 瞬时高峰。
- **零内存流式写入机制**：采用 `RNFetchBlob.fs.writeStream` 流式写入技术，实现边下载边写入，彻底规避 Base64 数据在大文件合并时的内存堆积问题。
- **异常上报与诊断增强**：重构下载异常捕获链路，现在每一个 TS 片段的下载失败都会记录具体的错误原因（如 HTTP 403、网络超时、解密失败等），并在日志中详细标注。
- **下载目录优化**：TS 缓存文件下载到本地 APP 免权限区域（`/Android/data/com.supertv.app/files/download`），无需申请额外存储权限。

### 优化
- **下载稳定性提升**：引入 30s 网络超时保护，防止单一片段卡死整体下载进程。
- **合并阶段优化**：引入串行写入锁机制，确保在高并发下载环境下，TS 片段依然能按正确物理顺序高效刷入磁盘。
- **临时目录自动清理**：完善 `.supertv.tmp` 临时文件清理机制，确保即使下载中断也不会产生无效占位。

### 修复
- 修复大文件下载时内存栈溢出的潜在问题。

## [5.5.8.523] - 2026-05-17

### 新增
- 增加缓存视频详情窗口，支持查看单集下载进度。
- 增加缓存视频直接播放功能。

### 优化
- 采用 LunaTV 架构：彻底移除 ffmpeg-kit 二进制依赖，改用纯 JS/TS 流处理引擎。
- 优化 M3U8 缓存逻辑：支持并发分段下载并顺序合并，显著提升缓存速度。
- 本地播放优化：内置本地 HTTP 代理服务，支持 Range 请求，大幅提升离线视频播放的稳定性与拖动体验。
- 优化缓存管理页面，增加 5 秒轮询自动更新状态。
- 优化 TV 端 UI 交互体验。

### 修复
- 修复缓存列表状态不同步的问题。
